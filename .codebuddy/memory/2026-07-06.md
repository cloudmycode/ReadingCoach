## server/sql/db.sql 适配 MariaDB 5.5.64
将 MySQL 8 导出的 `server/sql/db.sql` 改造为兼容 MariaDB 5.5：
- `json` 列改 `longtext`（articles.attachment_paths、words.translation），并去掉 TEXT/LONGTEXT 上非法的 `DEFAULT NULL`。
- `datetime NOT NULL DEFAULT CURRENT_TIMESTAMP` 的建表列改成 `timestamp NOT NULL DEFAULT CURRENT_TIMESTAMP`（5.5 只有 TIMESTAMP 能用 CURRENT_TIMESTAMP，且每表仅一个）。
- 带 `ON UPDATE CURRENT_TIMESTAMP` 的 updated_at 改为 `timestamp NULL DEFAULT NULL`（5.5 不支持第二个自动时间戳，自动更新行为丢失，需应用层用 NOW() 维护）。
- verification_codes 有两个自动 timestamp，把 expires_at 改为 `datetime NOT NULL`（应用 insert 时已显式赋值），created_at 保留自动。
- `utf8mb4_0900_ai_ci`（MySQL8 专有）改 `utf8mb4_unicode_ci`。
- 头部加 `SET SQL_MODE = '';`。
同步修复运行时建表：`server/internal/services/stats_service.go` 的 user_study_logs 表（同样的 datetime CURRENT_TIMESTAMP 问题），last_active_at 改 `DATETIME NOT NULL`（insert 已提供），created_at 自动 timestamp，updated_at `TIMESTAMP NULL DEFAULT NULL`。

## 服务器部署（宝塔）
服务器 45.79.40.29 用宝塔：nginx vhost 目录 `/www/server/panel/vhost/nginx/`，代码 `/www/wwwroot/reading_coach`（仓库根，Go 代码在其 `server/` 子目录，go.mod 在此）。部署方案：① 本地交叉编译静态二进制 `GOOS=linux GOARCH=amd64 CGO_ENABLED=0 go build -o reading-coach-server ./cmd/server` → scp 到服务器（规避服务器 Go 版本）；② 服务器 `.env` 里 MYSQL_PORT=3306（本机直连，不用隧道）；③ systemd 服务 reading-coach，WorkingDirectory=/www/wwwroot/reading_coach/server，EnvironmentFile=同目录/.env；④ nginx 走宝塔面板「反向代理」到 http://127.0.0.1:8080，加 client_max_body_size 20m + proxy_read_timeout 120s。
依赖检查结论：go.mod 要求 **go 1.24.4**（服务器编译需≥此版本，故建议本地编译）。运行时唯一外部依赖是 MySQL/MariaDB（已就绪）。**Redis 不需要**（internal/database/redis.go 的 NewRedisClient 是死代码，无处调用，config 无 REDIS）。图片处理用 disintegration/imaging + nfnt/resize，纯 Go，**不需要 libvips/imagemagick**；无 os/exec 无 CGO，CGO_ENABLED=0 静态编译实测通过。DeepSeek(api.deepseek.com)/微软TTS 为功能性依赖，需服务器出网，不填 key 则功能禁用但服务正常启动。
