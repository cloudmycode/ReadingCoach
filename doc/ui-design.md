# ReadingCoach UI 设计方案

## 1. 设计目标

ReadingCoach 的 UI 需要同时满足三个目标：

1. 让用户快速进入学习状态
2. 让句子理解和音频播放足够清晰
3. 让复习和坚持反馈具有正向激励感

整体风格应避免复杂和花哨，突出“专注、干净、可信赖”的学习体验。

## 2. 设计原则

### 2.1 低学习成本

首次打开就能理解主功能，拍照学习入口必须足够突出。

### 2.2 内容优先

学习页以句子内容为核心，减少多余视觉干扰。

### 2.3 明确反馈

上传、识别、解析、音频生成都必须有明确状态反馈。

### 2.4 复习导向

首页和历史页都要加强“昨天学过什么”“今天该复习什么”的引导。

## 3. 信息架构

建议采用底部 Tab：

1. 首页
2. 历史
3. 统计
4. 我的

拍照学习按钮可作为首页主入口，也可采用浮动主按钮形式强调。

## 4. 视觉方向

## 4.1 风格关键词

1. 清爽
2. 安静
3. 学习感
4. 轻专业

## 4.2 配色建议

主色：

- 深蓝 `#1F4E79`

辅助色：

- 青绿 `#3FA796`
- 浅米白 `#F8F6F1`
- 深灰 `#24323D`
- 中灰 `#667784`
- 浅灰 `#E6ECF2`

状态色：

- 成功 `#2E9E5B`
- 警示 `#F4A340`
- 错误 `#D85C4A`

整体避免高饱和度刺激色，保证长时间阅读舒适。

## 4.3 字体建议

iOS 原生可优先使用 `SF Pro`，中文界面搭配系统字体即可。重点通过字重和层级，而不是依赖过多颜色。

## 5. 页面设计方案

## 5.1 首页

### 目标

让用户一眼看到：

1. 如何开始新的学习
2. 今天应该复习什么
3. 今天已经完成了什么

### 模块布局

1. 顶部欢迎区
2. 主操作卡片
3. 昨日复习卡片
4. 最近学习列表
5. 今日数据摘要

### 关键元素

主操作卡片建议使用大卡片设计：

- 标题：`拍照开始学习`
- 副文案：`把英语阅读文章变成逐句讲解和听力练习`
- CTA：`立即拍照`

昨日复习卡片建议突出：

- `昨天学习了 2 篇，今天听一遍更容易记住`

## 5.2 拍照页

### 目标

降低拍摄失败率，让用户自然对准文章区域。

### 设计要点

1. 使用明显但不过度抢眼的取景框
2. 提供“请只拍正文区域”的轻提示
3. 拍完立刻进入预览确认页

### 预览页元素

1. 图片预览
2. 裁剪与旋转按钮
3. 重拍按钮
4. 上传并解析按钮

## 5.3 处理中页

### 目标

减少等待焦虑，让用户知道系统正在做什么。

### 状态展示建议

使用步骤进度组件：

1. 上传图片
2. 提取文章文字
3. 拆分句子并翻译
4. 生成朗读音频

### 设计建议

1. 使用柔和动效，不要使用强跳动 Loading
2. 提供预计耗时说明，如“通常需要 10 到 20 秒”

## 5.4 学习页

### 目标

成为产品体验核心，兼顾精读与复习。

### 页面结构

1. 顶部标题栏
2. 学习模式切换
3. 全文概览或段落摘要
4. 句子卡片列表
5. 底部播放控制条

### 句子卡片设计

每张卡片建议包含：

1. 句子编号
2. 英文原句
3. 中文翻译
4. 拆句讲解
5. 单句播放按钮
6. 收藏或标记按钮

### 视觉层级建议

1. 英文原句字号最大
2. 中文翻译次级
3. 拆句讲解采用浅背景信息块
4. 当前播放句子使用描边或浅色高亮

### 模式切换

建议提供两个模式标签：

1. `精读模式`
2. `复习模式`

精读模式默认显示翻译和讲解。  
复习模式默认隐藏中文，仅在点击后展开。

## 5.5 历史页

### 目标

帮助用户快速找到之前学过的文章。

### 列表项建议

1. 文章标题
2. 学习日期
3. 句子数
4. 最近播放状态
5. 快速复习按钮

### 筛选维度

1. 全部
2. 最近 7 天
3. 已收藏
4. 未复习

## 5.6 统计页

### 目标

让用户清楚看到自己在坚持，并愿意继续使用。

### 核心模块

1. 连续学习天数大卡
2. 今日学习篇数
3. 累计学习篇数
4. 最近 7 天趋势图
5. 月度打卡日历

### 设计建议

1. 连续学习天数放在首屏最显眼位置
2. 趋势图样式简单，避免金融图表感过重
3. 打卡日历强调“连续性”和“完成感”

## 5.7 我的页面

### 模块建议

1. 用户头像与昵称
2. 会员状态
3. 播放设置
4. 提醒设置
5. 缓存管理
6. 帮助与反馈

## 6. 关键交互说明

## 6.1 句子播放交互

1. 点击单句播放后，该句进入高亮状态
2. 全文播放时，列表自动滚动到当前句
3. 用户手动点击其他句子时，中断当前播放并切换

## 6.2 翻译显隐交互

1. 精读模式默认展开
2. 复习模式默认折叠
3. 用户可全局切换是否显示中文

## 6.3 解析失败交互

1. 明确告诉用户失败原因
2. 提供重试入口
3. 提供“编辑识别文字后重新解析”的备用方案

## 7. 空状态与异常状态

## 7.1 首页空状态

文案建议：

`拍下第一篇英语阅读文章，开始你的逐句理解训练。`

## 7.2 历史空状态

文案建议：

`你还没有学习记录，今天先完成一篇吧。`

## 7.3 统计空状态

文案建议：

`连续学习会在这里留下痕迹。`

## 7.4 网络异常

1. 提示当前网络状态异常
2. 保留已缓存文章可继续查看
3. 提供重新上传或稍后再试

## 8. 动效建议

1. 首页卡片进入时做轻微上浮渐显
2. 当前播放句子做柔和高亮过渡
3. 解析步骤状态切换使用简洁进度动画

动效应以低干扰为原则，避免影响学习专注度。

## 9. 无障碍建议

1. 支持动态字体
2. 提高播放按钮点击热区
3. 保证文本对比度
4. 支持 VoiceOver 朗读关键按钮标签

## 10. 设计交付物建议

设计师可按以下顺序输出：

1. 低保真线框图
2. 高保真视觉稿
3. 组件库
4. 核心交互动效稿
5. iPhone 标准尺寸标注稿

## 11. 后续可扩展设计

1. 生词收藏页
2. AI 问答页
3. 学习计划页
4. 会员订阅页

建议在 V1 先保证拍照学习和学习页体验足够扎实，再扩展外围页面。
